# Chapter 7: Exceptional Proposition Tree

## Core Idea

The Tractatus decimal tree is a powerful but rare authoring mode. It should be used only when absolute logical sharpness matters more than normal readability.

## Frameworks Introduced

- **Decimal tree / hypertext reading**
  - When to use: exceptionally hard logical problems, high-stakes prompts, formal conceptual design, dense theoretical arguments, or claims where dependencies must be inspectable.
  - How: create cardinal propositions; place every subordinate proposition under the exact proposition it clarifies; avoid flat sequence when dependency matters.
- **Ask-before-rigor rule**
  - When to use: before imposing this structure on user-facing writing.
  - How: ask the user whether they want Tractatus-level sharpness; warn that it may be too rigid for ordinary professional writing.
- **Dependency map**
  - When to use: when arguments are being confused by order, hidden assumptions, or mixed levels.
  - How: list top-level claims, then recursively attach definitions, evidence, exceptions, and implications.

## Key Concepts

- **Cardinal proposition**: a root claim that governs a branch.
- **Subordinate proposition**: a clarification, condition, or consequence of a parent claim.
- **Logical tree**: a hierarchy of dependencies, not a decorative outline.
- **Hypertext reading**: reading by conceptual dependency rather than simple page order.
- **High-rigor mode**: a rare mode that prioritizes logical inspection over natural prose.

## Mental Models

- The tree is for absolute sharpness, not everyday fluency.
- Do not use the tree to make ordinary writing look profound.
- Use the tree when hidden dependency is the actual problem.
- Convert back to normal prose once the argument is understood, unless the tree remains mandatory scaffolding.

## Anti-patterns

- **Tree everywhere**: applying proposition hierarchy to daily writing, emails, normal docs, or most professional prose.
- **Outline cosplay**: numbering sections without real dependency.
- **Rigidity as style**: using decimal form to create authority instead of clarity.
- **Flattened hard argument**: forcing a dependency-heavy argument into ordinary prose too early.
- **Unasked rigor**: imposing high-rigor mode without checking whether the user wants it.

## Key Takeaways

1. Default to normal clear prose.
2. Use proposition hierarchy only for exceptionally hard logical tasks.
3. Always ask before applying Tractatus-level sharpness.
4. A real tree encodes dependency, not importance alone.
5. Parent propositions should be understandable before their comments.
6. A tree can function as scaffolding; it may be kept if readers need it.
